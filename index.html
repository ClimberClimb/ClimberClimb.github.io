使用方法

JsonRPCMethod Class

首先urllib3里面的管理可以让你知道你需要所需要维持的connection pools

然后下面的id函数主要是返回根据md5方法名和调用时间来决定特定的id值。利用hashlib

最多的就是call方法,主要涉及的是利用json模块的json.dumps()把data数据转化为json格式的字符串,然后把从RPC Server返回的json字符串转化成python的obj等

JsonRPCClient class

主要是getattr来返回上面的JsonRPCMethod的实例对象,如果client没有这个方法属性就创建一个方法对象

Selector class

这个class主要是创建输入给安卓设备的参数,继承的字典dict，每个field对应一个有一个自己的mask，然后又有独立的总_mask,来判断这个node里面哪些field是设置的就或一个1 进去，删除则相反

  • clone函数来复制一个和自己一样的Selector
  • selector child来增加一个child的数量
  • sibling来增加一个sibling的数量

Adb Class

封装了adb的一系列的操作：

  • adb 函数主要是返回了操作系统下的adb程序的路径,主要利用了os模块下的os.path.join 和 os.path.exists等，其中
  • cmd和raw_cmd主要是下发打开adb进程，来进行一些命令操作
    -下面的主要是对于devices，forward，和一些操作的封装，比较有意思的就是列表向dict的转换

Not FoundHandler Class

这个主要是个descriptor，暂时还不是很清楚具体作用，先占着坑

Descriptors are objects with any of get, set, or delete. These descriptor objects can be used as attributes on other object class definitions.

AutomatorServer Class

包含前面的Adb Class的对象，封装了，push 文件，install apk, sdk version等操作。
比较关键的是这三个函数，对调用上面的RPCMethod，进行了一些封装操作，,调用jsonrpc_wrap返回的是JsonRPCClient的对象,而调用__jsonrpc的则少了添加了method，

当然server肯定有自己的start和stop方法，start方法里面主要涉及到的有是安装apk作为server，还是利用uiautomator里面的runtest来进行操作，取决于安卓的版本是多少,不过这里有个

AutomatorDevice Class

这个主要是对Android Device来进行封装，里面包含了前面的Automator Server对象，方便在Device里面直接和Server进行交互。

AutomatorDeviceUiObject Class && AutomatorDeviceObject Class

主要是封装了一些元素，来支持对某一个元素的各种操作，如点击等
